# About Modifiers

Modifiers provide a set of default classes that can be used within HTML templates. To avoid clashing with other classes and or frameworks that may be in use in the same project as Bristol fashion. The modifiers start with `_`.

The syntax of modifiers in bristol is `_utility-name:property(value)`

- utility: main name space such as flex, text, content, align
- property: top, bottom, left, right
- value: auto, 0-9

Colors are not included in the modifiers section. They may be added in the future. The problem comes with apply the color to the background or border etc. Additionally if we define these with their tones such as neutral in the HTML it makes switching these problematic. We suggest you do use them for you project but as part of you design vocabulary such as `._color:text(primary)` and `._color:text(secondary)` You can then apply borders and background to panels/cards etc.

export const parentStyle = {
  backgroundColor: 'rgb(207,216,220)',
  height: '7rem',
  padding: '0.5rem',
  margin: '0 0 1rem'
}

export const childStyle = {
  backgroundColor: 'rgb(55,71,79)',
  margin: '0.25rem',
  padding: '0.5rem',
  color: 'rgb(207,216,220)',
  borderRadius: '0.25rem',
  fontFamily: '"Noto serif", sans-serif',
  lineHeight: '1',
}

import './content/content.less'

<Story name="Default">
  {() => {
    return (
      <div>

      </div>
    )
  }}
</Story>

These can then be used in HTML

```
<div class="_content-center">
  <!--content-->
</div>
```

Or if you prefer you can use them as a mixin in Less

```
.element {
  ._content-center;
}
```

With Sass/Scss you unfortunately cannot pull in classes as mixins like this.
